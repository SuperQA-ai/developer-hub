The Continuous Integration Developer exam tests your knowledge and skills of the SuperQA Integration Delivery module.  

## Prerequisites

- Basic terminal skills
- Basic understanding of on-premise or cloud architecture

## Exam Details 

Exam Duration: 90 minutes <br/>
Question Type: Multiple choice

| Covered Domain                                              | % of Coverage |
| ------------------------------------------------------------| --------------|
| 1. Understanding of CI/CD Concepts and Best Practices        | 12%           |
| 2. SuperQA CI Pipeline Creation and Optimization             | 11%           |
| 3. Utilization and Management of SuperQA Connectors and Integrations | 11%       |
| 4. Implementing and Managing Docker and Containerization in SuperQA CI | 11%  |
| 5. SuperQA CI Configuration and Codebase Management          | 11%           |
| 6. Authentication and Security Management in SuperQA CI      | 11%           |
| 7. Implementing and Utilizing Test Intelligence in SuperQA CI | 11%           |
| 8. Advanced CI Features and Troubleshooting in SuperQA       | 11%           |
| 9. Implementation of Plugins and Extensions in SuperQA CI    | 11%           |

## Exam Objectives

<details>
<summary>List of Objectives</summary>

The following is a detailed list of exam objectives:

| #   | Objective |
|-----|-----------|
| **1**   | **Understanding of CI/CD Concepts and Best Practices** |
| 1.1 | Distinguish between Continuous Integration, Continuous Delivery, and Continuous Deployment |
| 1.2 | Identify key concepts related to CI/CD pipelines and their steps |
| 1.3 | Recognize best practices for implementing CI/CD in software development workflows |
| **2**   | **SuperQA CI Pipeline Creation and Optimization** |
| 2.1 | Construct a SuperQA CI pipeline and establish its stages and steps |
| 2.2 | Diagnose and resolve common errors and issues within the CI pipeline |
| 2.3 | Implement strategies to optimize pipeline execution and minimize build times |
| **3**   | **Utilization and Management of SuperQA Connectors and Integrations** |
| 3.1 | Configure and manage various SuperQA Connectors (e.g., Docker, Git) |
| 3.2 | Understand and implement integration with different platforms and OS |
| **4**   | **Implementing and Managing Docker and Containerization in SuperQA CI** |
| 4.1 | Establish Docker connectivity and manage Docker connectors |
| 4.2 | Develop and optimize Dockerfile configurations and image handling |
| 4.3 | Diagnose and mitigate common Docker-related issues and errors |
| **5**   | **SuperQA CI Configuration and Codebase Management** |
| 5.1 | Understand and implement the configuration of SuperQA CI Codebase |
| 5.2 | Optimize and manage code repository, connectors, and trigger settings |
| **6**   | **Authentication and Security Management in SuperQA CI** |
| 6.1 | Configure and manage authentication methods for various connectors (e.g., Git, Docker) |
| 6.2 | Implement and utilize Secrets Management tools with SuperQA CI |
| 6.3 | Understand and implement access control using SuperQA RBAC configurations |
| **7**   | **Implementing and Utilizing Test Intelligence in SuperQA CI** |
| 7.1 | Configure and synchronize SuperQA Test Intelligence with main/master branches |
| 7.2 | Implement strategies to improve and optimize test execution times |
| **8**   | **Advanced CI Features and Troubleshooting in SuperQA** |
| 8.1 | Implement and utilize advanced features like caching and conditional executions in CI |
| 8.2 | Diagnose, troubleshoot, and resolve common issues and errors in CI stages |
| **9**   | **Implementation of Plugins and Extensions in SuperQA CI** |
| 9.1 | Understand, install, and manage plugins in the SuperQA CI environment |
| 9.2 | Implement use-cases using various SuperQA CI and third-party plugins |

</details>
